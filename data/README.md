# 数据目录说明

## 目录结构

```
data/
├── train/                    # 训练集（80%的数据）
│   ├── T恤/                 # 每个类别一个文件夹
│   │   ├── img001.jpg       # 图片文件
│   │   ├── img002.jpg
│   │   └── ...
│   ├── 衬衫/
│   ├── 卫衣/
│   ├── 外套/
│   ├── 牛仔裤/
│   ├── 休闲裤/
│   ├── 运动鞋/
│   └── 皮鞋/
│
└── validation/               # 验证集（20%的数据）
    ├── T恤/
    ├── 衬衫/
    └── ...（结构同上）
```

## 数据要求

### 图片格式
- 支持格式: JPG, JPEG, PNG
- 推荐尺寸: 224x224 或更大（系统会自动调整）
- 文件大小: 建议每张小于5MB

### 数据量
- **最少**: 每类50张（训练40张 + 验证10张）
- **推荐**: 每类100-500张（训练80-400张 + 验证20-100张）
- **理想**: 每类1000+张

### 数据质量
- ✓ 图片清晰，主体明显
- ✓ 背景简洁，避免复杂背景
- ✓ 类别标注准确
- ✓ 每个类别的数量尽量平衡
- ✗ 避免模糊、遮挡、过暗的图片

## 数据收集方法

### 方法1: 下载公开数据集（推荐）

1. **Fashion-MNIST**（入门级）
   - 简单的服装数据集
   - 适合快速测试

2. **DeepFashion**（专业级）
   - 网址: http://mmlab.ie.cuhk.edu.hk/projects/DeepFashion.html
   - 大量真实服装图片
   - 需要申请下载

3. **Kaggle数据集**
   - Fashion Product Images
   - 网址: https://www.kaggle.com/datasets/paramaggarwal/fashion-product-images-dataset

### 方法2: 自己收集

1. **网络搜索**
   - Google Images
   - 电商网站（注意版权）
   - 免费图库

2. **拍摄实物**
   - 拍摄自己的衣服
   - 拍摄商店展示
   - 注意光线和角度

3. **数据增强**
   - 系统会自动进行数据增强
   - 包括旋转、翻转、缩放等
   - 可以将100张扩展到数千张

## 数据分配

### 训练集 vs 验证集

- **训练集**: 用于训练模型（80%）
- **验证集**: 用于评估模型（20%）

### 分配示例

如果你有100张T恤图片：
- 80张放入 `data/train/T恤/`
- 20张放入 `data/validation/T恤/`

## 下一步

1. 将图片放入对应的类别文件夹
2. 确保每个类别都有足够的图片
3. 运行训练脚本:
   ```bash
   python train_easy.py
   ```
